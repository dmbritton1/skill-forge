# SkillForge v0.2 — Slice A Design: Substrate (Ledger, Trust, Attribution)

*Approved design from brainstorming, 2026-07-10. Parent spec: skillforge-architecture-v4.md (draft 0.7), Sections 4.3, 6, 9.2, 11.2, 13.*

## v0.2 decomposition (context)

v0.2 ("the loop closes") is built as four sequential, independently shippable slices:

- **Slice A (this document)** — Substrate: SQLite event ledger, local trust registry + quarantine + `/review`, trust-gated native sync, distiller attribution artifacts.
- **Slice B** — Retrieval & tiering: BM25 index, UserPromptSubmit injection hook (token budget, session dedupe, trust gate), `/find`, hot-tier fixed budget. Depends on A.
- **Slice C** — Outcome tracking: PostToolUse symptom/verification matching, symptom-triggered anti-skill injection, marker protocol (`skillforge-usage` engine skill), injection-time fingerprint snapshots, Stop reconciler, three-bucket confidence. Depends on A + B.
- **Slice D** — Validation, capture, stats: Tier A gate (executable + critique), Stop-hook capture suggestions, `/stats`. Depends on A.

## Slice A scope

In: ledger, trust registry, `/skillforge:review`, SessionStart sync hook, distiller upgrade (fingerprints + verification.command), native-dir gitignore migration.

Out (later slices): retrieval, injection, confidence buckets, symptom matching, Tier A validation, holdouts, `/stats`.

## Design decisions made during brainstorming

1. **Trust gates native loading via a SessionStart sync hook** (not save/review-time-only materialization, not deferred). Rationale: the threat is content-you-didn't-author arriving via `git pull`/clone — any repo, not just teammates — and "modification re-quarantines" (§11.2) requires an automatic check, not a manual one. Solo benefits: native copies become derived/rebuildable cache with one write-through point; multi-machine self-use works coherently.
2. **Committed project native copies are a v0.1 security leftover and must be migrated out**: `.claude/skills/skillforge-hot/` becomes gitignored, existing committed copies removed from tracking.
3. **Aggregates are SQL views over an events table**, never stored columns (events can't be reconstructed from aggregates; parent spec §9.2 calls this ordering load-bearing).
4. **Trust hashes cover the exact bytes that get materialized** (CRLF-normalized, no stripping). Mutable ledger state lives only in `ledger.db`, never in SKILL.md, so hash stability under bookkeeping comes from state separation, not strip-at-hash-time — and any file change, including injected `status:`/`confidence:` lines, re-quarantines (§11.2). (Amended after final review: the original strip-then-hash decision let content evade the integrity hash.)
5. **`verification.command` is required frontmatter for `kind: skill`; `fingerprints` is warning-only.** Library is currently empty, so no backfill path is needed.

## Components

### 1. Ledger — `scripts/ledger.py`, `~/.claude/skillforge/ledger.db`

SQLite, WAL mode (concurrent hook writers). One `events` table:

```sql
CREATE TABLE IF NOT EXISTS events (
  id INTEGER PRIMARY KEY,
  event_type TEXT NOT NULL,      -- save | review | injection | detection | holdout | verdict
  skill TEXT NOT NULL,           -- skill name (kebab-case id)
  session TEXT,                  -- session id if known
  turn INTEGER,
  tier TEXT,                     -- hot | warm | cold (B+)
  trigger TEXT,                  -- prompt | symptom | manual (B/C)
  detection TEXT,                -- marker | verification | fingerprint | judge | holdout (C+)
  preexisting_fingerprint INTEGER, -- 0/1 (C+)
  outcome TEXT,                  -- success | failure | blocked | approved | quarantined ...
  ts TEXT NOT NULL               -- ISO-8601 UTC
);
```

Aggregates as views (created in Slice A, mostly consumed later): `skill_aggregates(skill, uses, successes, failures, injections, last_used)`.

Interface: importable (`log_event(**fields)`, `connect()`) and CLI (`ledger.py log --event-type save --skill NAME ...`, `ledger.py show <skill>`). Slice A writers: `save_skill.py` (event_type `save`), `trust.py` (event_type `review`).

### 2. Trust registry — `scripts/trust.py`, `~/.claude/skillforge/trust.json`

```json
{ "<skill-name>": { "hash": "<sha256>", "approved_ts": "...", "origin": "self|reviewed" } }
```

- Hash = SHA-256 of the CRLF-normalized file content — exactly the bytes sync materializes; any change re-quarantines.
- `save_skill.py` auto-trusts at save (`origin: self`) — the local user watched the skill get created (§11.2).
- CLI: `trust.py check <file>` → `trusted | quarantined | modified`; `trust.py list-quarantined [--project-root DIR]`; `trust.py approve <file>` (records hash, `origin: reviewed`, logs review event).
- `trust.json` is never committed and lives only in the global store.

### 3. Sync — `scripts/sync.py`, SessionStart hook (`hooks/hooks.json`)

The only writer of native dirs. For every skill in both stores (global + project):

- trusted and hash-unchanged → ensure native copy exists and is current
- unknown or hash-changed → remove native copy if present; count as quarantined

Prints one line to hook output only when quarantines exist: `skillforge: N skill(s) quarantined pending /skillforge:review`. Budget <100ms typical. `save_skill.py` calls sync's materialize step directly after save so self-saves stay immediate.

Migration: repo `.gitignore` gains `.claude/skills/skillforge-hot/`; any committed native copies are `git rm --cached`-ed.

### 4. Distiller upgrade — both engine skills + `save_skill.py` validation

- `skills/distilling-skills/SKILL.md`: contract step to emit `fingerprints:` (2–3 distinctive code fragments — distinctive means "would not appear in unrelated code"; `express.raw({type: 'application/json'})` qualifies, `npm install` does not) and `verification.command` (the machine-runnable form of the `## Verification` section, emitted directly in frontmatter, never parsed from prose).
- `skills/distilling-failures/SKILL.md`: same, with fingerprints taken from the `## Fix` section.
- `save_skill.py`: reject `kind: skill` drafts missing `verification.command`; print a warning (not reject) when `fingerprints` is absent or has fewer than 2 entries.

### 5. `/skillforge:review` — `commands/review.md`

Prompt-content command: run `trust.py list-quarantined`, then for each quarantined file show the user its full content (untrusted content — the model may summarize alongside, but the user sees the real text and makes the call), and on explicit per-skill approval run `trust.py approve <file>`. Never batch-approve. A reviewed pulled skill starts as advisory-status-stripped content earning its own ledger history (§11.2).

## Testing

Stdlib-assert test files (no pytest), sandbox HOME pattern from v0.1:

- `tests/test_ledger.py` — insert events, aggregate view math, two concurrent writers (WAL).
- `tests/test_trust.py` — auto-trust on save; unknown file quarantined; modified file re-quarantined; hash stability under mutable-frontmatter changes.
- `tests/test_sync.py` — trusted materialized, quarantined evicted, changed-hash evicted; idempotence.
- E2E task: hook fires on session start (verified via `claude --plugin-dir ... -p`), save→trust→sync round trip in sandbox HOME.

## Non-goals / deferred within v0.2

Retrieval and injection (B), confidence buckets and all detection layers (C), Tier A validation and capture suggestions and `/stats` (D), per-author signing and PR review queues (team tier, v1.0).
